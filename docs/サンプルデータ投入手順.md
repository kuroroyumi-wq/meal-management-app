## サンプルデータ（本番）投入手順

本番で「レシピ登録〜調理指示書表示」が実装どおり動くかを確認するために、サンプルの食材・レシピを一括投入します。

### 注意（重要）

- **`SUPABASE_SERVICE_ROLE_KEY` は最強権限**です。漏洩すると危険なので、共有やコミットは絶対にしないでください。
- この手順は **ローカル端末から本番DBへ直接投入**します（RLSをバイパスできます）。

---

## 1) 事前準備

### A. DBにカラム追加が済んでいること

Supabase SQL Editor で `supabase/migrations/005_add_recipe_cutting_notes.sql` を実行して、成功していること。

### B. 本番 Supabase の Service role key を取得

Supabase Dashboard → Project Settings → API → **Project API keys** → **service_role** をコピーします。

---

## 2) 実行（ローカルから）

プロジェクトルートで次を実行します。

```bash
# 本番Supabase URL（既に .env.local に入っているなら不要）
export NEXT_PUBLIC_SUPABASE_URL="https://xxxx.supabase.co"

# 本番の service_role key（絶対に漏らさない）
export SUPABASE_SERVICE_ROLE_KEY="xxxxx"

npm run seed:samples
```

成功すると、作成したレシピIDが表示されます。

---

## 3) 画面で確認（本番）

1. レシピ一覧を開く
2. 追加されたレシピ（例：肉じゃが、味噌汁など）が見えること
3. レシピ詳細で「調理指示書」として以下の順で表示されること
   - 材料
   - 切り方・大きさの目安（入力済みのもののみ）
   - 作り方・手順（入力済みのもののみ）
   - 参考URL（あれば）
