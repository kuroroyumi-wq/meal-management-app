# 要件定義書
## 高齢者施設向け 食事管理Webアプリ

**プロジェクト名**: Meal Management System  
**作成日**: 2026年3月  
**目的**: ポートフォリオ用フルスタックWebアプリ開発

---

## 1. プロジェクト概要

### 1.1 背景

高齢者施設では、毎日多人数分の食事を提供する必要がある。現場では以下の問題がある。

- 献立作成に時間がかかる
- 人数変更による材料計算が手作業
- 在庫管理がアナログ（スプレッドシート）
- 食材ロスが発生
- 栄養バランス管理が難しい

これらの業務をデジタル化し、効率化するための食事管理Webアプリケーションを開発する。

### 1.2 システムの目的

1. 献立作成の効率化
2. 人数に応じた材料自動計算
3. 食材在庫の可視化
4. 食材原価の管理
5. 栄養バランスの管理
6. AIによる献立作成支援

## 4. 機能要件（全11項目）

### 要件① 材料自動計算
- 人数入力で材料量を自動計算
- 計算式: 必要量 = 基準材料量 × 人数

### 要件② 週間献立作成
- 1週間分の献立を作成・管理
- 日付×食事区分（朝・昼・夜）で管理

### 要件③ 原価計算
- レシート入力で材料単価・原価を計算
- 計算式: 材料量 × 単価

### 要件④ 在庫管理
- 毎日の使用量と仕入れから在庫を把握
- 余った食材を可視化

### 要件⑤ レシピスクレイピング
- レシピURLから材料データを自動取得
- 対応サイト: デリッシュキッチン等

### 要件⑥ AI献立自動生成
- 材料データを元に1週間献立を自動作成
- 直近4週間の献立は除外

### 要件⑦ 余り食材活用
- 余った材料から献立を再作成
- 1週間パターンを10種類作成

### 要件⑧ スタッフ作業管理
- シフト別タスク管理
- LINE通知（またはアプリ通知）

### 要件⑨ 栄養管理
- 献立クリックで1人前の栄養素表示
- 栄養士が栄養情報を登録

### 要件⑩ 月次統計
- 売上・食材ロス・原価の月次集計
- 食事提供価格: 朝食200円・昼食500円・夕食500円

### 要件⑪ 販売管理
- フリーズドライ・チルド販売管理
- 主食200円・副菜100円（5食程度販売予定）
