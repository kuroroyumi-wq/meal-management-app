### 2. Supabase の RLS を有効化（セキュリティ・推奨）

**RLS（Row Level Security）** とは、テーブルごとに「誰がどの行にアクセスできるか」をデータベース側で制限する機能です。有効にすると、**ログインしていないリクエスト**ではデータの取得・追加・更新・削除ができず、**ログイン済みユーザー**だけがアプリのデータを操作できるようになります。

#### 実行順序（重要）

RLS は **既にあるテーブル** に対してしか有効にできません。次の順で実行してください。

| 順番 | ファイル | 内容 |
|------|----------|------|
| 1 | `001_initial_schema.sql` | 食材・レシピ・献立・在庫・スタッフなどのテーブル作成 |
| 2 | `002_rls_policies.sql` | 上記テーブルに対する RLS 有効化とポリシー作成 |
| 3 | `003_sales.sql` | 販売用テーブル作成と RLS（販売だけ後から追加した場合） |

**「ingredients が存在しません」** と出た場合は、**1 をまだ実行していない**か、別プロジェクトの DB を見ている可能性があります。まず 1 を実行してください。

#### 具体的な手順

1. **Supabase ダッシュボードを開く**
   - ブラウザで https://supabase.com/dashboard を開く
   - このプロジェクト（meal-management）を選択する

2. **SQL Editor を開く**
   - 左メニューで **「SQL Editor」** をクリック
   - **「New query」** をクリックして、新しいクエリ用の入力欄を表示する

3. **002_rls_policies.sql の内容をコピーする**
   - ローカルの **`supabase/migrations/002_rls_policies.sql`** を開く
   - ファイルの中身を **すべて選択**（Cmd+A / Ctrl+A）して **コピー**（Cmd+C / Ctrl+C）

4. **SQL Editor に貼り付けて実行する**
   - Supabase の SQL Editor の入力欄をクリックし、**貼り付け**（Cmd+V / Ctrl+V）
   - 右下の **「Run」**（または Cmd+Enter）をクリック

5. **結果を確認する**
   - 画面上部に **「Success. No rows returned」** のようなメッセージが出れば成功です。
   - エラーが出た場合：
     - **「policy "..." already exists」** → すでに同じ名前のポリシーが存在します。RLS は有効な状態なので、そのままで問題ありません。
     - **「relation "ingredients" が存在しません」**（42P01）→ **テーブルがまだ作成されていません**。先に **`001_initial_schema.sql`** を実行してから、改めて `002_rls_policies.sql` を実行してください（下記「実行順序」参照）。

#### 実行後の挙動

- **ログイン済み**: これまでどおり、食材・レシピ・献立などすべての画面と API が利用できます。
- **未ログイン**: 保護された API を叩いても、Supabase がポリシーで弾くため、データは返らずエラーになります。画面はミドルウェアで `/login` にリダイレクトされます。
