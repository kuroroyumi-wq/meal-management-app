# デザイン統一ルール

高齢者施設向け食事管理アプリの UI を統一し、品質を保つためのルールです。  
新規画面・コンポーネント追加・デザイン改善時はこのルールに従ってください。

---

## 1. カラー

### 1.1 役割と使い分け

| 用途 | 指定方法 | 使用例 |
|------|----------|--------|
| **プライマリ** | `bg-primary` / `text-primary` / `border-primary` | メインアクション、サイドバーアクティブ、リンク・強調 |
| **プライマリ前景** | `text-primary-foreground` | プライマリ背景上の文字 |
| **破壊的操作** | `bg-destructive` / `text-destructive` / `variant="destructive"` | 削除ボタン、エラー |
| **ニュートラル** | `text-zinc-700`（重要ラベル）, `text-zinc-600`（補足）, `text-zinc-500`（弱い補足） | 本文・ラベル・説明 |
| **成功** | 緑系（Toast・Alert success） | 保存完了、成功メッセージ |
| **エラー** | `Alert variant="error"`（赤背景） | バリデーション・API エラー |

### 1.2 背景・枠

- ページ背景: `bg-background`（body 継承）
- カード・パネル: `bg-white dark:bg-zinc-800/80`
- 枠線: `border-zinc-200 dark:border-zinc-700`
- 区切り線: `border-zinc-200 dark:border-zinc-700`

### 1.3 禁止事項

- プライマリ以外のブランド色を増やさない（必要なら CSS 変数で定義）
- インラインで `#hex` を乱用しない（セマンティックな Tailwind / CSS 変数を使う）

---

## 2. タイポグラフィ

| 要素 | クラス | 用途 |
|------|--------|------|
| ページタイトル | `text-2xl font-bold text-zinc-900 dark:text-zinc-50` | 各ページの h1 |
| セクション見出し | `text-lg font-semibold text-zinc-900 dark:text-zinc-50` | フォーム・カード内の h2 |
| サブセクション・ラベル | `text-sm font-medium uppercase tracking-wide text-zinc-600 dark:text-zinc-400` | 「概要」「メニュー」などの見出し |
| 本文 | `text-base`（body 継承） | 説明文・リスト |
| ラベル（フォーム） | `text-sm font-medium text-zinc-700 dark:text-zinc-300` | input の label |
| 補足・キャプション | `text-sm text-zinc-500 dark:text-zinc-400` | カード下の説明 |
| 数値・KPI | `text-2xl font-bold tabular-nums text-zinc-900 dark:text-zinc-50` | ダッシュボードの数値 |

- 高齢者向けのため、**本文は `text-base`（16px）以上**を維持する。
- 入力欄はモバイルで 16px 以上（globals.css で指定済み）。

---

## 3. スペーシング

| 用途 | 指定 | 例 |
|------|------|-----|
| ページタイトル下 | `mb-2`（タイトル）, `mb-8`（説明文まで） | ダッシュボード・一覧 |
| セクション間 | `mb-6` 〜 `mb-10` | フォームとテーブルの間 |
| セクション見出しと内容 | `mb-4` | 「概要」と KPI グリッド |
| カード・グリッド間 | `gap-4` | grid gap |
| フォーム内 | `gap-3`（グリッド）, `mt-6`（ボタンエリア） | 食材登録フォーム |
| ボタン間 | `gap-2` または `gap-3` | キャンセル・保存 |
| 一覧テーブル | `px-4 py-3`（セル）, `divide-y` | テーブル |

- 8px ベースで揃える: `2, 3, 4, 6, 8`（Tailwind の 4 = 1rem など）。

## 4. 角丸・影

| 要素 | 指定 |
|------|------|
| カード・パネル | `rounded-xl` |
| ボタン | `rounded-lg`（Button コンポーネント既定） |
| 入力欄 | `rounded-md` |
| バッジ・ピル | `rounded-full` |
| カードの影 | `shadow-sm`、hover 時 `shadow-md` |

## 5. コンポーネント別ルール

### 5.1 ボタン

- **メインアクション**: `variant="default"`（プライマリ色）、`min-h-11 min-w-11` でタップ領域を確保。
- **副次アクション**: `variant="outline"`。
- **破壊**: `variant="destructive"`（削除など）。
- **送信ボタン**: 右寄せ（`ml-auto`）、キャンセルは左。送信中は `disabled` と「保存中...」表示。

### 5.2 カード（クリック可能）

- `DashboardCard` を使用。`href` でリンク化。
- スタイル: `rounded-xl border border-zinc-200 bg-white p-5 shadow-sm`、hover で `hover:scale-[1.02] hover:shadow-md`。
- フォーカス: `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`。

### 5.3 フォーム

- **ラッパー**: カード風に `rounded-xl border border-zinc-200 bg-white p-6 shadow-sm`。
- **見出し**: `text-lg font-semibold`、`mb-4`。
- **ラベル**: `block text-sm font-medium text-zinc-700 dark:text-zinc-300`、必須は「必須」または `*` と `aria-required="true"`。
- **入力**: `rounded-md border border-zinc-300 px-3 py-2 text-base focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring`。
- **ボタン行**: `mt-6 flex flex-wrap gap-3`、キャンセル左・保存右。

### 5.4 テーブル

- **ラッパー**: `overflow-x-auto rounded-lg border border-zinc-200 dark:border-zinc-700`。
- **ヘッダ**: `bg-zinc-100 dark:bg-zinc-800`、`px-4 py-3 font-medium`。
- **行**: `bg-white dark:bg-zinc-900`、hover で `hover:bg-zinc-50 dark:hover:bg-zinc-800/80`。
- **セル**: `px-4 py-3`、文字は `text-base` または `text-sm` で統一。
- **操作列**: ボタンは `min-h-10 min-w-10` でタップしやすく。

### 5.5 空状態（EmptyState）

- **アイコン**: lucide-react、`size-10`、`text-zinc-400`。背景は `rounded-full bg-zinc-100 dark:bg-zinc-800`。
- **タイトル**: `text-lg font-semibold`。
- **説明**: `text-base text-zinc-600 dark:text-zinc-400`、`max-w-sm`。
- **CTA**: プライマリの Button / Link、`min-h-11`。

### 5.6 トースト・アラート

- **成功トースト**: 固定 `fixed right-4 top-4`、緑系、`rounded-lg`、約 4 秒で非表示。
- **エラー**: `Alert variant="error"`。インラインで表示。

### 5.7 サイドバー

- **アクティブ**: `bg-primary text-primary-foreground`、左ボーダー `border-l-4 border-primary`。
- **非アクティブ**: `text-zinc-700 dark:text-zinc-300`、hover で背景変化。
- **タブレット（md〜lg）**: 幅 `w-16`、アイコンのみ。`lg` 以上で `w-56` とラベル表示。
- **ナビ項目**: `min-h-[44px]`、アイコン + `<span className="hidden lg:inline">` でラベル。

---

## 6. レスポンシブ

| ブレークポイント | 用途 |
|------------------|------|
| **default** | スマホ（1 列・ドロワー） |
| **md: (768px)** | タブレット（2 列グリッド・サイドバーアイコンのみ） |
| **lg: (1024px)** | PC（3 列グリッド・サイドバー全文表示） |

- グリッド: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` を基本とする。
- テーブル: 横スクロールを許容（`overflow-x-auto`）、`min-w-[400px]` などで可読性を確保。
- main 余白: `p-6 pt-14 md:pt-6`（スマホでハンバーガー分の pt）。

---

## 7. アクセシビリティ

- **フォーカス**: インタラクティブ要素は `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` を付与。
- **タップ領域**: ボタン・ナビは最低 44px（`min-h-11` など）。
- **ラベル**: 入力には `label` と `id` を対応させる。必須は `aria-required="true"`。
- **エラー**: `role="alert"` または `aria-live="polite"` を必要に応じて付与。

---

## 8. 禁止・非推奨

- ページごとにバラバラの `rounded-*` や `shadow-*` を増やさない（上記ルール内で統一）。
- インラインスタイルや `style={{}}` は極力使わない（Tailwind / コンポーネントで対応）。
- デザインだけのために関数コンポーネントを細かく分割しすぎない（再利用する場合のみコンポーネント化）。

---

## 9. 変更時の確認

- 新規・変更後は **ビルド**（`npm run build`）で型・構文エラーがないこと。
- 既存画面で **フォントサイズ・余白・色** がルールから外れていないか確認。
- 必要に応じてこのドキュメントを更新し、ルールを一元化する。
