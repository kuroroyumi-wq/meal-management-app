# デプロイ・セットアップ手順

このドキュメントでは、**Supabase で販売テーブルを作る手順**と **Vercel へのデプロイ手順**を説明します。

---

## 1. Supabase で販売管理テーブルを作成する

### 1-1. Supabase ダッシュボードを開く

1. ブラウザで [https://supabase.com/dashboard](https://supabase.com/dashboard) を開く
2. ログインして、このプロジェクト（食事管理アプリ用）を選択する

### 1-2. SQL Editor を開く

1. 左メニューで **「SQL Editor」** をクリック
2. **「New query」** をクリックして、新しいクエリ用の入力欄を開く

### 1-3. 003_sales.sql の内容を貼り付けて実行

1. ローカルの次のファイルを開く  
   `supabase/migrations/003_sales.sql`
2. **ファイルの中身をすべてコピー**（Ctrl+A → Ctrl+C / Cmd+A → Cmd+C）
3. Supabase の SQL Editor の入力欄に **貼り付け**（Ctrl+V / Cmd+V）
4. 右下の **「Run」**（または「実行」）をクリック
5. 画面上部に **「Success. No rows returned」** のような成功メッセージが出れば完了

#### エラーが出た場合

- **「relation "sales_products" already exists」**  
  → すでにテーブルが作成済みです。そのままで問題ありません。
- **「permission denied」**  
  → ログインしている Supabase のアカウントで、そのプロジェクトのオーナーまたは管理者か確認してください。

## 2. Vercel にデプロイする

### 2-1. リポジトリを GitHub に push する

まだ GitHub にコードを上げていない場合:

1. [GitHub](https://github.com) で新しいリポジトリを作成（例: `meal-management-app`）
2. ターミナルでプロジェクトのフォルダに移動し、次を実行:

```bash
cd /Applications/cursorフォルダ/meal-management-app

# まだ git 初期化していない場合
git init
git branch -M main

# リモートを追加（URL は自分のリポジトリに書き換え）
git remote add origin https://github.com/あなたのユーザー名/meal-management-app.git

# コミットして push
git add .
git commit -m "Prepare for Vercel deploy"
git push -u origin main
```

すでにリモートがある場合は、変更をコミットして push するだけです:

```bash
git add .
git commit -m "Prepare for Vercel deploy"
git push origin main
```

### 2-2. Vercel でプロジェクトをインポートする

1. ブラウザで [https://vercel.com](https://vercel.com) を開く
2. **「Sign Up」** または **「Log in」** でログイン（GitHub 連携がおすすめ）
3. トップ画面で **「Add New…」** → **「Project」** をクリック
4. **「Import Git Repository」** で、先ほど push した **meal-management-app** のリポジトリを選択
5. **「Import」** をクリック

### 2-3. 環境変数を設定する

「Configure Project」の画面で:

1. **「Environment Variables」** のところを開く
2. 次の変数を **1つずつ** 追加する（名前と値を入力して **Add**）:

| Name | Value | 備考 |
|------|--------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | あなたの Supabase の Project URL | Supabase → Settings → API で確認 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | あなたの Supabase の anon public キー | 同上 |
| `ANTHROPIC_API_KEY` | あなたの Anthropic API キー | AI 献立生成を使う場合のみ |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE チャネルアクセストークン | LINE 通知を使う場合のみ（任意） |

3. **「Production」「Preview」「Development」** のどれに効かせるか選ぶ（通常は 3 つともチェックで OK）
4. 設定が終わったら **「Deploy」** をクリック

### 2-4. デプロイ完了を確認する

1. ビルドとデプロイが終わるまで 1〜3 分ほど待つ
2. **「Congratulations」** と表示され、URL（例: `https://meal-management-app-xxx.vercel.app`）が表示されたら成功
3. その URL を開き、**ログイン画面**（`/login`）が表示されることを確認
4. Supabase の Authentication で作成したユーザーでログインできるか確認

### 2-5. 本番 URL を Supabase に登録する（ログインする場合に必要）

ログイン後にリダイレクトされる URL を Supabase に教える必要があります。

1. Supabase ダッシュボード → **Authentication** → **URL Configuration**
2. **「Redirect URLs」** に、Vercel の本番 URL を追加する  
   例: `https://meal-management-app-xxx.vercel.app/**`
3. **「Save」** をクリック

これで、本番環境からもログイン・リダイレクトが正しく動くようになります。
