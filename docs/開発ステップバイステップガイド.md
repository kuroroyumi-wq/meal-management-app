## 準備：GitHub に設計書を保存する

### やること
プロジェクトに `docs/` フォルダを作り、設計書を保存してGitHubにアップする。

### フォルダ構成
```
meal-management-app/
├── docs/
│   ├── requirements.md      ← 要件定義書
│   ├── basic-design.md      ← 基本設計書
│   ├── er-diagram.md        ← DB設計（テーブル定義SQL）
│   └── cursor-context.md   ← Cursor用コンテキスト
└── README.md
```

### 専門用語
- **リポジトリ**: GitHubに保存するプロジェクトの「箱」。コードの変更履歴も記録される
- **Push**: ローカル（自分のPC）の変更をGitHubにアップロードすること

### コマンド（Cursorのターミナルで実行）
```bash
git add docs/
git commit -m "docs: add design documents"
git push origin main
```

## Phase 1：環境セットアップ（Day 1）

### STEP 1：Next.jsプロジェクトを作成する

**場所**: Cursorを開いて、上部メニュー「Terminal」→「New Terminal」

**専門用語**
- **Next.js**: Reactをベースにした人気のWebフレームワーク。画面表示とサーバー処理を1つのプロジェクトで作れる
- **TypeScript**: JavaScriptに「型」を追加した言語。バグを事前に防ぎやすい
- **App Router**: Next.js 14の新しいファイル構成方式

```bash
npx create-next-app@latest meal-management-app \
  --typescript --tailwind --eslint --app \
  --src-dir=false --import-alias="@/*"
cd meal-management-app
```

> ⚠️ 途中で質問が出たらすべてEnterキーを押す（デフォルトでOK）

---

### STEP 2：必要なパッケージをインストールする

**専門用語**
- **パッケージ（ライブラリ）**: 他の人が作った便利な部品。`npm install` で追加できる
- **Supabase**: データベースを無料で使えるクラウドサービス。ログイン機能も付いている
- **shadcn/ui**: 見た目が整ったUIコンポーネント集。ボタンやテーブルをすぐに使える

```bash
npm install @supabase/supabase-js @supabase/ssr
npm install @anthropic-ai/sdk
npm install cheerio axios
npm install lucide-react clsx tailwind-merge
npx shadcn@latest init
```

---

### STEP 3：Supabaseプロジェクトを作成する

**場所**: ブラウザで作業

**専門用語**
- **PostgreSQL**: 世界で広く使われているデータベースの種類
- **anon key（匿名キー）**: フロントエンドからSupabaseに接続するための公開キー

**手順**
1. https://supabase.com にアクセス
2. 「Sign Up」でアカウント作成（GitHubアカウントでOK）
3. 「New Project」→ プロジェクト名を入力（例: meal-management）
4. 作成完了後、左メニュー「Settings」→「API」を開く
5. 「Project URL」と「anon public」キーをコピーしておく

---

### STEP 4：.env.localファイルを作成する

**場所**: Cursorでプロジェクトのルート（一番上のフォルダ）に新規ファイル作成

**専門用語**
- **環境変数**: パスワードやAPIキーなどの秘密情報をコードに直接書かずに管理する仕組み
- **.env.local**: 環境変数を書くファイル。GitHubには絶対アップロードされない（自動で除外）

**ファイル名**: `.env.local`（ドットから始まる名前に注意）

```env
NEXT_PUBLIC_SUPABASE_URL=ここにSTEP3でコピーしたURLを貼り付け
NEXT_PUBLIC_SUPABASE_ANON_KEY=ここにSTEP3でコピーしたkeyを貼り付け
ANTHROPIC_API_KEY=後でAnthropicのサイトから取得して追加
```

> ⚠️ このファイルは絶対にGitHubにPushしないこと。.gitignoreに自動で追加されているので通常は問題ない

---

### STEP 5：データベースのテーブルを作成する

**場所**: Supabaseのダッシュボード（ブラウザ）

**専門用語**
- **テーブル**: データベースの「表」のこと。Excelのシートに似ている
- **SQL**: データベースを操作するための言語。`CREATE TABLE`でテーブルを作る
- **マイグレーション**: データベースの構造を変更・作成する作業のこと

**手順**
1. Supabaseダッシュボード → 左メニュー「SQL Editor」をクリック
2. 「New query」をクリック
3. `docs/er-diagram.md` に書かれているSQLを全部コピーして貼り付け
4. 右上の「Run」ボタンをクリック
5. 「Success」と表示されればOK

## Phase 2：Cursorで実装開始（Day 2〜）

### STEP 1：設計書をCursorのAIに渡す

**場所**: Cursorの右側のチャット欄（AIアシスタント）

**専門用語**
- **コンテキスト**: AIに渡す「背景情報」。これを渡すとAIの回答精度が大幅に上がる
- **プロンプト**: AIへの指示文のこと

**Cursorのチャットに以下をコピペして送信**:
```
以下の設計に従ってこのプロジェクトを実装してください。

【プロジェクト概要】
高齢者施設向け食事管理Webアプリ（Next.js14 + Supabase + Claude API）

【技術スタック】
- Next.js 14 App Router + TypeScript
- Tailwind CSS + shadcn/ui
- Supabase（PostgreSQL）
- Anthropic Claude API

まず以下を作成してください：
1. lib/supabase/client.ts（ブラウザ用Supabase接続）
2. lib/supabase/server.ts（サーバー用Supabase接続）
3. types/index.ts（TypeScript型定義）
```

---

### STEP 2：食材マスタのAPI（バックエンド）を作成

**専門用語**
- **API（エーピーアイ）**: フロントエンド（画面）とバックエンド（サーバー）が通信するための窓口
- **Route Handler**: Next.jsでAPIを作る仕組み。`app/api/` フォルダ内に作る
- **CRUD**: Create（作成）・Read（読取）・Update（更新）・Delete（削除）の略

**Cursorに送信**:
```
ingredientsテーブルのCRUD APIをNext.js Route Handlerで作成してください。

GET    /api/ingredients        → 食材一覧取得
POST   /api/ingredients        → 食材新規登録
PUT    /api/ingredients/[id]   → 食材更新
DELETE /api/ingredients/[id]   → 食材削除

TypeScriptで型安全に実装してください。
```

---

### STEP 3：食材管理の画面（フロントエンド）を作成

**専門用語**
- **コンポーネント**: 画面のパーツ（ボタン、テーブルなど）を部品として作ったもの
- **Server Components**: サーバー側で実行されるNext.jsの画面。データ取得が速い

**Cursorに送信**:
```
食材一覧ページを作成してください。
場所: app/ingredients/page.tsx

要件：
- shadcn/uiのTableコンポーネントを使用
- 表示項目：食材名・単位・単価・カテゴリ
- 登録・編集・削除ボタン付き
- Tailwind CSSでスタイリング
```

## 完成：デプロイ・ポートフォリオ公開（Week 7）

### STEP 1：Vercelにデプロイ

**専門用語**
- **デプロイ**: 作ったアプリをインターネット上に公開すること
- **Vercel**: Next.jsと相性抜群の無料ホスティングサービス。GitHubと連携すると自動デプロイされる

**手順**
1. https://vercel.com にアクセス → GitHubでログイン
2. 「Add New Project」→ GitHubのリポジトリを選択
3. 「Environment Variables」にenv.localの内容を入力
4. 「Deploy」ボタンをクリック
5. 数分後にURLが発行される（例: https://meal-management-app.vercel.app）
